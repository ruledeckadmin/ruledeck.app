How it works

Four steps.
Up and running today.

STEP 01 · IT

Deploy in under 10 minutes

Your IT team runs two commands: create a .env file with your credentials and docker compose up. Ruledeck starts, validates the licence, and is accessible in your browser. No build steps, no dependencies to install.

STEP 02 · BUSINESS

Define your logic as a table

Open Ruledeck in any browser. Add condition columns for the inputs your decision depends on — a customer tier, an order value, a risk score, a product type. Add result columns for what you want back. Each row is one rule. No formulas, no code, no IT ticket.

STEP 03 · BUSINESS

Test, then publish with one click

Use the built-in testbench to run your rule against real inputs before going live. Debug mode shows exactly which row matched and why. When satisfied, press Publish. Ruledeck instantly creates a live REST API endpoint. Roll back any prior version in seconds.

STEP 04 · BUSINESS

Iterate without engineering

Policy changed? New tier to add? Threshold to tighten? Open the rule in your test environment, edit the table, verify in the testbench, and publish. The update is live in seconds. Every change is logged in the audit trail.

Data & Security

Your data
stays yours.

Unlike SaaS decision platforms, Ruledeck runs entirely within your own environment. No decision data, customer information, or business logic is ever sent to an external server.

  • Self-hosted deployment

    Install on your own server, data centre, or private cloud. You control the infrastructure and the data.

  • API key + session authentication

    Solve endpoints require a valid API key. The admin UI requires session authentication with brute-force lockout after 5 failed attempts.

  • Hardened by default

    Helmet security headers, HttpOnly/SameSite cookies, rate limiting, body size enforcement, and SQL injection protection ship enabled out of the box.

  • No usage telemetry

    Ruledeck does not collect usage data or analytics. No decision data, business logic, or customer information is ever transmitted to any external server.

  • GDPR / data sovereignty

    Data never leaves your servers, making regional data-residency compliance straightforward for regulated industries.
